RiskAssessment

  • Potential outcomes for a subject with likelihood

    Base StructureDefinition for RiskAssessment Resource

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

       Field Card. Type and Description
        basedOn 0..1 Resource
      A reference to the request that is fulfilled by this risk assessment.
        basis 0..* List<Resource>
      Indicates the source data considered as part of the assessment (FamilyHistory, Observations, Procedures, Conditions, etc.).
        code 0..1 Concept
      The type of the risk assessment performed.
        condition 0..1 Condition
      For assessments or prognosis specific to a particular condition, indicates the condition being assessed.
        context 0..1 Encounter | EpisodeOfCare
      The encounter where the assessment was performed.
        identifier 0..1 Identifier
      Business identifier assigned to the risk assessment.
        method 0..1 Concept
      The algorithm, process or mechanism used to evaluate the risk.

      (Example)
        mitigation 0..1 String
      A description of the steps that might be taken to reduce the identified risk(s).
        note 0..1 Annotation
      Additional comments about the risk assessment.
        occurrence[x] 0..1 DateTime | interval<DateTime>
      The date (and possibly time) the risk assessment was performed.
        parent 0..1 Resource
      A reference to a resource that this risk assessment is part of, such as a Procedure.
        performer 0..1 Practitioner | Device
      The provider or software application that performed the assessment.
        prediction 0..* List<Prediction>
      Describes the expected outcome for the subject.
        reason[x] 0..1 Concept | Resource
      The reason the risk assessment was performed.
        status 1..1 code
      The status of the RiskAssessment, using the same statuses as an Observation.

      Binding: ObservationStatus (Required)
        subject 0..1 Patient | Group
      The patient or group the risk assessment applies to.
