Schedule

  • A container for slot(s) of time that may be available for booking appointments

    Base StructureDefinition for Schedule Resource

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

       Field Card. Type and Description
        active 0..1 Boolean
      Whether this schedule record is in active use, or should not be used (such as was entered in error).
        actor 1..1 Patient | Practitioner | RelatedPerson | Device | HealthcareService | Location
      The resource this Schedule resource is providing availability information for. These are expected to usually be one of HealthcareService, Location, Practitioner, Device, Patient or RelatedPerson.
        comment 0..1 String
      Comments on the availability to describe any extended information. Such as custom constraints on the slot(s) that may be associated.
        identifier 0..* List<Identifier>
      External Ids for this item.
        planningHorizon 0..1 interval<DateTime>
      The period of time that the slots that are attached to this Schedule resource cover (even if none exist). These cover the amount of time that an organization's planning horizon; the interval for which they are currently accepting appointments. This does not define a "template" for planning outside these dates.
        serviceCategory 0..1 Concept
      A broad categorisation of the service that is to be performed during this appointment.

      Binding: ServiceCategory (Example)
        serviceType 0..* List<Concept>
      The specific service that is to be performed during this appointment.

      Binding: ServiceType (Example)
        specialty 0..* List<Concept>
      The specialty of a practitioner that would be required to perform the service requested in this appointment.

      Binding: Practice Setting Code Value Set (Preferred)
