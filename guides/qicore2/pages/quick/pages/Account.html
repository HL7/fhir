Account

  • A resource with narrative, extensions, and contained resources

    Base StructureDefinition for Account Resource

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

       Field Card. Type and Description
        active 0..1 interval<DateTime>
      Indicates the period of time over which the account is allowed to have transactions posted to it. This period may be different to the coveragePeriod which is the duration of time that services may occur.
        balance 0..1 Money
      Represents the sum of all credits less all debits associated with the account. Might be positive, zero or negative.
        coverage 0..* List<Coverage>
      The party(s) that are responsible for payment (or part of) of charges applied to this account (including self-pay). A coverage may only be resposible for specific types of charges, and the sequence of the coverages in the account could be important when processing billing.
        coveragePeriod 0..1 interval<DateTime>
      Identifies the period of time the account applies to; e.g. accounts created per fiscal year, quarter, etc.
        currency 0..1 Code
      Identifies the currency to which transactions must be converted when crediting or debiting the account.

      Binding: value set (Example)
        description 0..1 String
      Provides additional information about what the account tracks and how it is used.
        identifier 0..* List<Identifier>
      Unique identifier used to reference the account. May or may not be intended for human use (e.g. credit card number).
        name 0..1 String
      Name used for the account when displaying it to humans in reports, etc.
        owner 0..1 Organization
      Indicates the organization, department, etc. with responsibility for the account.
        status 0..1 code
      Indicates whether the account is presently used/useable or not.

      Binding: AccountStatus (Required)
        subject 0..1 Patient | Device | Practitioner | Location | HealthcareService | Organization
      Identifies the patient, device, practitioner, location or other object the account is associated with.
        type 0..1 Concept
      Categorizes the account for reporting and searching purposes.
