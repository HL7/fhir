Identifier Systems

Responsible Owner: [%wgt vocab%] Work GroupStandards Status:Normative

Identifier systems are URIs that may be used in the system element of the Identifier datatype.

In the past, this page included a list of registered URIs. However, that content was migrated to HL7 Terminology which provides a registry of Identifier systems that is actively curated by HL7 International. If a URI is defined in HL7 Terminology, it SHOULD be used in preference to any other Identifier.system. For example, all system URIs defined on HL7 Terminology prior to the release of FHIR R6 are expected to be used in systems claiming conformance to FHIR R6.

Many identifier systems are 'local', for example the identifiers used for employee numbers, local medical record numbers, local order and encounter numbers, etc. The system URIs for such identifiers are generally determined and managed by the issuing system. However, 'shared' identifiers that are used across a wide variety of systems, such as license numbers, government-assigned identifiers, etc. need to have consistent URIs. In most cases, the assigners of these identifiers will not have published an official URI.

If an identifier system for a 'shared' identifier is not listed in HL7 Terminology, implementers SHOULD (and HL7 international published specifications SHALL) follow the process for determining Identifier.system URIs defined by the HL7 Terminology Services Management Group (TSMG), as documented in the Terminology Expectations in FHIR policy.

See the list of code systems that can be used in the system element of the Coding datatype.
