Using FHIR Resources with hData

hData describes the use of a RESTful architecture to manage a set of granular resources in a coherent record structure. hData and FHIR share the same basic RESTful transport (the RESTful specification is a profile on the OMG hData RESTful transport - specific link when one is available)

hData builds on top of that a set of resources and addresses issues that provide the architecture to maintain a coherent record, such as a patient care record. This page documents how FHIR resources may be used for the components of an hData record architecture.
