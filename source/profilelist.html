Profiles defined as part of FHIR

A Profile is a special resource that provides additional rules about how a type of resource is utilized in a particular context of use, or for a particular use case.

Some particular use cases are common or important enough to be described as a part of the specification itself:

Name Usage
General
General Extensions A set of general extensions defined for the convenience of implementers that use FHIR resources
Element Extensions A set of extensions that constrain data elements, whether used in DataElement, Profiles or Questionnaires
US Profile Contains constraints and extensions to the FHIR specification specific to the US realm
ISO 21090 Extra ISO 21090 features not included in data types because of core design policy
Rendering Extensions Extensions that govern the appearance of data elements when rendered for human display or printing
Tools Extensions Extensions defined by the build tools. These extensions are candidates for being elevated into the core in a subsequent version
DataElement
ISO 11179 Data Element Profile A profile showing how to use DataElement to express 11179 Data_Element and Concept_Elements. At present, the profile is a partially-complete place-holder
Structured Data Capture Data Element Sets expectations for data elements registered or used as part of the structured data capture project
Questionnaire/QuestionnaireAnswers
Questionnaire Core extensions for Questionnaire
Structured Data Capture (SDC) Form Design Contains constraints and extensions to the Questionnaire resource for form designs compliant with the SDC implementation guide
Structured Data Capture (SDC) Data Capture Contains constraints and extensions to the QuestionnaireAnswers resource for questionnaire data capture compliant with the SDC implementation guide
<%profilelist%>

Other profiles may be found in the general profile repository (reference to be provided).
