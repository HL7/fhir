FHIR Change History

Changes from the DSTU ballot (v0.11) to the DSTU final copy (0.20?):

  • ResourceReference: remove "type" element - it had no actual function
  • Bundle - add the "fhir-base" link that SHALL be present if relative references are found in the bundle
  • JSON - rename "_id" to "id", and change the way that primitive types are represented (split id and extension out into _[name])
  • Documents & Messages: document that these are marked with tags to identify them
  • Extensions: make rules around obligations for exchange
  • REST API: The syntax of the URLs has changed (remove @ from the ids). resources are no longer returned on create/update/transaction, and rules around the transction response have changed. CORS is now recommended (and in the conformance resource).
  • Searching: clarify that searching is not just for rest. Extend _sort parameter. Add relevance score, and sorting by that. Add _profile parameter
  • CodeableConcept: move primary to Coding.primary
  • Coding: add primary, valueset and version & align code system version rules with core principles
  • Choice: remove choice and add Coding.valueset
  • ValueSet: add define.version, and a new constraint that codes must be unique in the definition space, and tighten up publication (maps and cross links, expansions, deal with poly-heirarchies & deprecated codes, publish all v3 code systems, map the data types), add more uniqueness variants
  • The following pages have been rewritten (Editorial changes only): "Resource Definitions", "Using codes" and the overview page
  • FamilyHistory: change the value set for relation type to use the v3 value set directly
  • Remove Coverage resource
  • ImagingStudy: add link to DiagnosticOrder. Trim mappings to DICOM. Limit URL to WADO-RS only (not WADO)
  • DocumentReference: aligned definitions more closely with XDS. Worked over the XDS Profile
  • DocumentManifest: added this from scratch to round out XDS support
  • Document: revised status list per o-o analysis of reporting cycles
  • Validation: added Profile tags, and explained how they are used for validation
  • Standardised identifiers: AdverseReaction, AllergyIntolerance, CarePlan, Condition, FamilyHistory, Procedure, Questionnaire all have identifier 0..*
  • Condition & Procedure: several changes around relatedItem
  • Procedure: made type a required element, changed type and cardinality for indications and complications, and updated definition of procedure
  • CarePlan: made patient optional, and reworked to link activites to goals and goals to concerns, and allow other resource kinds for activities
  • Alert: improved the definitional documentation
  • Message: make the event code a Coding (also in the conformance statement too)
  • ImmunizationProfile - rename to ImmunizationRecommendation
  • Profile: Remove the concept of re-useable bindings, and collapse binding in line to element definition. Change definition.type.bundled to definition.type.aggregation, and add more invariants about uniqueness
  • Conformance: Add Conformance.profile. Move system operations to rest.operations from rest.resource.operations, & add updateCreate flag
  • Add Informative ConceptMap resource
