Pharmacy Work Group Changes

Dosage / DosageDetails structure

  • Dosage.simple removed (FHIR-54651)
    A simple dosage is now expressed as a single step. The simple element has been removed as it was redundant. The dosdet-1 invariant has been replaced: if only one step is present, DosageDetails.step.safety shall not be populated. Documentation and examples updated accordingly.
  • DosageDetails.renderedInstruction is now required (1..1) (FHIR-54014)
    renderedInstruction must always be present to ensure the full dosage intent is communicated in every case. Definition and comment updated.
  • Dosage.site is now CodeableReference(BodyStructure) (FHIR-54848)
    Dosage.site can now reference a BodyStructure for richer site descriptions including laterality and location qualifiers. The same coded binding is retained.

Dosage examples

  • step.component structure corrected (FHIR-56016)
    Examples now correctly place timing and doseAndRate under step.component rather than directly under step.
  • Structural representation corrected to match text (FHIR-54523)
    Fixed mismatches between narrative and structure (e.g. Example 1 doseQuantity was 10mg instead of 2.5mg) and cross-format inconsistencies across YAML, JSON, and XML (missing doses, wrong units, wrong safety periods).

MedicationStatement

  • Reason vs relatedClinicalInformation clarified (FHIR-54451)
    .reason covers what leads to the medication being taken (or not taken). .relatedClinicalInformation is for information relevant to the statement but not the actual indication — that belongs in .reason.

MedicationStatement / MedicationRequest

  • Entered-in-error status definitions corrected (FHIR-54903)
    The entered-in-error status now simply indicates that the resource was entered in error and is not valid.
